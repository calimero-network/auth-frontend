Auth Flows

AdminFlow, ApplicationFlow, PackageFlow — the three scenarios handled

Flow Selection

App.tsx calls useFlowDetection(), which reads URL query params (and localStorage fallback). Priority order:

PackageFlow
package-name query param is present. Fetches manifest from registry, installs app, then proceeds to permissions + optional context selection.
AdminFlow
permissions=admin or mode=admin in query params, or no other indicators (default fallback). No app installation.
ApplicationFlow
application-id query param is present. Verifies the app is installed on the node, then proceeds to permissions + optional context selection.

Single-context vs Multi-context Mode

Both ApplicationFlow and PackageFlow support two modes, set via the mode query param:

single-context
After permissions, the user picks a context (and identity) via ContextSelector. The context_id and context_identity are included in the callback hash. ContextSelector uses mero-ui Menu + MenuItem components for selection and a create-context form when no matching context exists.
multi-context
Context selection step is skipped entirely. The generated token has multi-context permissions. No context_id or context_identity in the callback hash — the calling app manages context routing itself.

Legacy detection: if permissions=application is in the URL (and no explicit mode), the flow defaults to multi-context. Otherwise single-context.

AdminFlow

Login is handled by EnsureAdminSession before this flow renders. AdminFlow itself only handles permission review and token generation.

1

EnsureAdminSession (pre-flow)

Checks calimero_access_token in localStorage. If valid, renders children immediately. If not, shows ProviderSelector → UsernamePasswordForm → calls mero.auth.generateTokens().

2

Review permissions

Renders PermissionsView showing the requested admin permissions with risk badges.

3

Generate token

Calls generateClientKeyDirect({ context_id: '', context_identity: '', permissions }) — a direct POST to /admin/client-key.

4

Redirect

Appends access_token and refresh_token to the callback-url as a URL hash and navigates.

ApplicationFlow

Triggered by application-id query param. Login is handled by EnsureAdminSession before this flow renders.

1

App install check

ApplicationInstallCheck verifies the application is installed on the node. Installs it from application-path if needed.

2

Review permissions

Renders PermissionsView.

3

Context selection (single-context mode only)

Renders ContextSelector — context list filtered to the target application ID. User picks a context and then an identity. If no matching context exists, offers a create-context form.

4

Generate token + redirect

Calls generateClientKeyDirect({ context_id, context_identity, permissions }). Redirects to callback URL with access_token, refresh_token, context_id, context_identity, node_url in hash.

PackageFlow

Triggered by package-name query param. Login is handled by EnsureAdminSession before this flow renders.

1

Manifest + install

ManifestProcessor fetches the manifest from the registry (or by direct URL). If app is already installed on the node, skips re-install. Otherwise calls mero.admin.installApplication({ url, metadata }). Stores application ID in localStorage.

2

Review permissions

Renders PermissionsView. Shows manifest app name/version in the header if available.

3

Context selection (single-context mode only)

Renders ContextSelector. Same as ApplicationFlow step 3.

4

Summary + confirm

Shows app identity, context ID, identity, and permissions for user confirmation before generating the token. Uses mero-ui Card with inline detail rows.

5

Generate token + redirect

Calls generateClientKeyDirect() with permissions scoped to the installed application ID (context:execute[app_id]). Redirects with access_token, refresh_token, application_id, context_id, context_identity, node_url in hash.
