Auth Flows

AdminFlow, ApplicationFlow, PackageFlow — the three scenarios handled

Flow Selection

App.tsx determines which flow to run based on URL query parameters or the presence of an application_id in the hash:

AdminFlow
No application_id present. Authenticates an admin session against the node without installing or launching any application.
ApplicationFlow
application_id present. Authenticates, then generates a client key for the given context+identity pair so the application can make signed RPC calls.
PackageFlow
package_id present. Authenticates, installs the package as an application, creates a context, and returns tokens + context ID to the opener.

AdminFlow

1

Check hash for SSO tokens

Calls parseAuthCallback(window.location.hash). If found, skips to step 4 — zero interaction needed.

2

Show LoginView

Renders username + password form. On submit, calls sdk.auth.login().

3

Token stored

Access + refresh tokens stored in TokenStore. SDK is now authenticated.

4

Redirect to node dashboard

Admin flow complete — redirects to the admin dashboard URL passed in query params.

ApplicationFlow

1

SSO check or login

Same as AdminFlow steps 1–3. If SSO tokens present, no login form shown.

2

Resolve context identity

Calls sdk.admin.getIdentities() to get the node's signing identity for this context.

3

Generate client key

Calls sdk.admin.generateClientKey({ contextId, contextIdentity }) (camelCase in mero-js v2). Returns a signing keypair the app will use for RPC.

4

Return tokens + key

Posts the client key and tokens to the opener window or redirects with hash params.

PackageFlow

1

SSO check or login

Authenticate via hash tokens or login form.

2

Install package

Calls sdk.admin.installApplication({ url, metadata }).

3

Create context

Calls sdk.admin.createContext({ applicationId }) with the newly installed app.

4

Generate client key + return

Same as ApplicationFlow steps 3–4. Returns all credentials to the opener.
