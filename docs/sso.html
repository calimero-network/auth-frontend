Session persistence via localStorage — how login is skipped when valid tokens already exist

Session Persistence

EnsureAdminSession is the gate for all three flows. On mount it checks localStorage for existing admin tokens and validates them before showing any UI:

Tokens present + valid

Children render immediately. No login screen shown. The user was already authenticated from a previous session.

Tokens present but expired

Tries mero.auth.refreshToken(). On success, updates localStorage and continues. On failure, clears tokens and shows login.

No tokens

Shows ProviderSelector → UsernamePasswordForm → calls mero.auth.generateTokens() → stores tokens → renders children.

localStorage Keys

EnsureAdminSession reads and writes these keys via helpers in src/lib/mero.ts:

calimero_access_token
Admin JWT. If present and valid (or refreshable), login is skipped.
calimero_refresh_token
Used to refresh the access token when it expires.
calimero_app_endpoint
Node base URL. Used to initialise the MeroJs instance.

Desktop / Programmatic Bypass

External tools (like Calimero Desktop) can bypass the login screen by pre-seeding localStorage before opening the auth-frontend URL:

// Set from the Desktop (same origin) before loading auth-frontend
localStorage.setItem('calimero_access_token', accessToken);
localStorage.setItem('calimero_refresh_token', refreshToken);
localStorage.setItem('calimero_app_endpoint', nodeUrl);

// When auth-frontend loads, EnsureAdminSession finds the tokens,
// validates them, and renders the flow directly — no login screen.

Security Notes

  • Same-origin only: localStorage is origin-scoped. A Desktop bypass only works if auth-frontend and the Desktop share the same origin.
  • Token expiry: Access tokens expire normally. EnsureAdminSession calls refreshToken() on load if needed — no re-login required.
  • Callback hash: Tokens sent back to calling apps are placed in the URL hash (never the query string) so they are not included in server access logs.
  • Hash cleared after read: Calling apps should clear the hash after reading (history.replaceState(null, '', location.pathname)) to avoid tokens appearing in browser history.
