SSO Integration

URL hash token bypass — skip auth when tokens arrive in the URL

What SSO Bypass Does

When Calimero Desktop opens an app window, it embeds valid JWT tokens in the URL hash. The auth frontend detects these tokens on load and skips the login, context-selection, and permission screens entirely — initialising directly with the provided credentials.

Without SSO

User sees: node URL input → credentials → context selection → permission grant → app loads. Takes 4–5 steps.

With SSO

User clicks "Open" in Desktop → app loads immediately. Zero extra steps. Tokens come pre-filled from the Desktop session.

Fallback

If no tokens are in the URL hash, the normal auth flow runs. SSO is purely additive — existing functionality unchanged.

Hash Parameters

The auth frontend reads these keys from window.location.hash:

access_token
JWT access token. Used immediately for authenticated node API calls.
refresh_token
JWT refresh token. Stored for session renewal when the access token expires.
node_url
Base URL of the Calimero node. Tells the SDK where to connect.
application_id
The application package ID. Used to scope context selection and permissions.

Reading Tokens in mero-js

mero-js exports parseAuthCallback() to safely parse all four hash parameters:

import { parseAuthCallback } from '@calimero-network/mero-js';

const authData = parseAuthCallback(window.location.hash);

if (authData) {
  // { accessToken, refreshToken, nodeUrl, applicationId }
  // Initialise SDK directly, skip auth flow
  await sdk.auth.initFromTokens(authData);
} else {
  // No hash tokens — run normal auth flow
}

Security Notes

  • Hash not sent to server: URL hash fragments are never included in HTTP requests. Tokens don't leak to server access logs.
  • Same-origin only: The Desktop only opens URLs for apps registered in the node's app registry, limiting the attack surface.
  • Token expiry: Access tokens expire on a normal schedule. The refresh token is also passed so the app can renew without re-authentication.
  • Hash cleared after read: Applications should clear the hash after reading (history.replaceState(null, '', location.pathname)) to avoid tokens appearing in browser history.
