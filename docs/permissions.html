Permissions

How client keys, identities, and access control work in the auth frontend

Identity Model

The Calimero node uses a public-key identity model. Each node has one or more root identities. Applications interact with contexts using client keys — ephemeral keypairs authorized against a root identity:

Root Identity

A permanent keypair held by the node. Used to authorize client keys. Listed via sdk.admin.getIdentities().

Client Key

A short-lived keypair for a specific context. Generated by generateClientKey(). Used to sign RPC calls.

Token Generation

Client key generation uses a direct fetch to /admin/client-key, not an SDK method. Fields are snake_case.

// generateClientKeyDirect() in src/lib/mero.ts
const response = await generateClientKeyDirect({
  context_id: contextId,       // '' for admin/multi-context flows
  context_identity: identity, // '' for admin/multi-context flows
  permissions: ['context:execute', 'context:list'],
});

// Returns: { access_token, refresh_token }
// These are scoped tokens for the calling app — not the admin session tokens.

What the App Receives

After the flow completes the app is redirected to its callback URL with these fields appended as a URL hash:

access_token
Scoped JWT for the calling app. Always present.
refresh_token
Long-lived token for renewing the access token. Always present.
node_url
The Calimero node base URL. Present in ApplicationFlow and PackageFlow.
context_id
The selected or created context ID. Present in single-context flows only.
context_identity
The selected identity public key for the context. Present in single-context flows only.
application_id
The installed application ID. Present in PackageFlow only.

Security Notes

Private key storage
The client private key should be stored in sessionStorage (cleared on tab close), not localStorage. Never send to any server.
Access token lifespan
Access tokens are short-lived. The auth frontend refresh flow exchanges the refresh token for a new access token automatically.
SSO security
When receiving tokens via URL hash from the desktop app, clear the hash immediately after reading to prevent token leakage in browser history.
