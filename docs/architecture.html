Architecture

React 19 flows — AdminFlow, ApplicationFlow, PackageFlow — backed by mero-js v2

Source Structure

src/App.tsx
Root. Calls handleUrlParams() to persist URL params to localStorage, then uses useFlowDetection() to determine which flow to render. All three flows are wrapped in EnsureAdminSession.
src/components/auth/EnsureAdminSession.tsx
Handles all login. Checks localStorage for existing tokens and validates them. If no valid token, shows ProviderSelector → UsernamePasswordForm → calls mero.auth.generateTokens(). Only renders children once a valid admin token is secured.
src/flows/AdminFlow.tsx
Shows PermissionsView for admin permission review, then calls generateClientKeyDirect() and redirects to callback URL with tokens in hash.
src/flows/ApplicationFlow.tsx
app-check → permissions → context-selection (single-context mode only) → redirect. Uses ApplicationInstallCheck to verify the app is installed on the node.
src/flows/PackageFlow.tsx
manifest → permissions → context-selection (single-context mode only) → summary → redirect. Uses ManifestProcessor to fetch + install from registry.
src/components/manifest/ManifestProcessor.tsx
Fetches manifest from registry by package name or manifest URL. Detects existing installation (skips re-install). Calls mero.admin.installApplication() and stores the returned application ID.
src/components/context/ContextSelector.tsx
Two-step picker: context list → identity list. Filters contexts by target application ID. Offers create-context form (protocol + init args JSON) when no matching context exists. Uses mero-ui components throughout.
src/components/permissions/PermissionsView.tsx
Renders per-permission risk cards using mero-ui and mero-tokens. Adapts button labels based on mode (admin vs application).
src/hooks/
Custom hooks: useFlowDetection (reads URL params, returns flow source + mode), useContextCreation (install app + create context via mero.admin.*), useContextSelection (fetch contexts and identities).
src/lib/mero.ts
Singleton MeroJs instance. Token helpers for localStorage. generateClientKeyDirect() — direct POST /admin/client-key that bypasses the SDK and returns access_token + refresh_token.

Flow State Machines

Each flow is a state machine. ApplicationFlow steps:

app-check
verify install
permissions
user approves
context
single-ctx only
done
redirect to app

The context-selection step only runs in single-context mode. In multi-context mode the flow goes straight to redirect after permissions — no specific context is locked in. See Auth Flows for mode details.

mero-js v2 Integration

The auth frontend uses @calimero-network/mero-js (replaces the older calimero-client package). Key integration points:

generateClientKeyDirect()
Client key generation bypasses the SDK entirely — it's a direct POST /admin/client-key with { context_id, context_identity, permissions } (snake_case). Returns { access_token, refresh_token }.
mero.auth.generateTokens()
Used by EnsureAdminSession to authenticate with username/password. Returns the admin session tokens stored in localStorage.
mero.admin.*
Context management: getContexts(), createContext({ protocol, applicationId, initializationParams }), getContextIdentitiesOwned(contextId). Application: installApplication({ url, metadata }), getLatestPackageVersion().

Tech Stack

React 19

Functional components, hooks, Suspense boundaries for loading states.

TypeScript

Strict mode. All API types imported from mero-js.

Vite

Fast HMR, optimised build output. Served as a static bundle by the node's web server.

mero-ui

@calimero-network/mero-ui — design system used across flows. Components: Button, Card, Stack, Flex, Menu, Text, EmptyState, etc.

Emotion

CSS-in-JS for component styles. Theme tokens for consistent design.

React Router DOM

Navigation between steps in multi-step flows.

pnpm

Package management. Lock file committed.
