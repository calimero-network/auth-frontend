Context Management

How the auth frontend creates, selects, and manages Calimero contexts

What is a Context?

A context is an isolated execution environment on a Calimero node. Each context runs one application (WASM binary) and has its own state, identity list, and access control. Applications interact with the node by sending RPC calls to a specific context ID.

The auth frontend creates or selects a context during the ApplicationFlow and PackageFlow — this is what enables the app to subsequently make signed RPC calls.

Context Selection Flow

1

List existing contexts

Calls sdk.admin.getContexts(). Filters to contexts that run the target application ID.

2

Select or create

If a matching context exists, uses it. If not (PackageFlow), creates one via sdk.admin.createContext({ applicationId }).

3

Generate client key

Calls sdk.admin.generateClientKey({ contextId, contextIdentity }) to produce a signing keypair for this context.

4

Return to app

The context ID, client public key, and private key are passed back to the calling application via postMessage or URL hash.

Context API Calls

// List all contexts
const contexts = await sdk.admin.getContexts();

// Get a specific context
const ctx = await sdk.admin.getContext('ctx_abc');

// Create a new context
const newCtx = await sdk.admin.createContext({
  applicationId: 'app_xyz',
});

// Generate client key (mero-js v2 — camelCase)
const key = await sdk.admin.generateClientKey({
  contextId: newCtx.id,
  contextIdentity: identity.publicKey,
});

Context Object Shape

interface Context {
  id: string;            // Context ID (opaque string)
  applicationId: string;// Installed application ID
  rootHash: string;     // Current state root hash
  members: string[];   // Identity public keys with access
}
