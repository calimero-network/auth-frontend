Context Management

How the auth frontend creates, selects, and manages Calimero contexts

What is a Context?

A context is an isolated execution environment on a Calimero node. Each context runs one application (WASM binary) and has its own state, identity list, and access control. Applications interact with the node by sending RPC calls to a specific context ID.

The auth frontend creates or selects a context during the ApplicationFlow and PackageFlow — this is what enables the app to subsequently make signed RPC calls.

Context Selection Flow

This step only runs in single-context mode. In multi-context mode it is skipped entirely.

1

List existing contexts

Calls mero.admin.getContexts(). Filters the list to contexts whose applicationId matches the target app. Rendered with mero-ui Menu + MenuItem components.

2

Select identity

Once a context is picked, calls mero.admin.getContextIdentitiesOwned(contextId) and shows the identity list.

3

Create context (if needed)

If no matching context exists, shows a create-context form. Calls mero.admin.createContext({ protocol, applicationId, initializationParams }). Protocol is currently fixed to near. Returns { contextId, memberPublicKey }.

4

Return to flow

The selected or created contextId and identity are passed back to the parent flow, which includes them in the generateClientKeyDirect() call and the final callback hash.

Context API Calls

// List all contexts (useContextSelection hook)
const res = await mero.admin.getContexts();
const contexts = res?.data?.contexts ?? res?.contexts ?? [];

// Get owned identities for a context
const iRes = await mero.admin.getContextIdentitiesOwned(contextId);
const identities = iRes?.data?.identities ?? iRes?.identities ?? [];

// Create a new context (useContextCreation hook)
const createRes = await mero.admin.createContext({
  protocol: 'near',
  applicationId: 'app_xyz',
  initializationParams: Array.from(new TextEncoder().encode(initArgs)),
});
const { contextId, memberPublicKey } = createRes?.data ?? createRes;

// Generate scoped token (direct fetch, not SDK method)
const tokens = await generateClientKeyDirect({
  context_id: contextId,         // snake_case
  context_identity: memberPublicKey,
  permissions: ['context:execute'],
});

Context Object Shape

interface Context {
  id: string;            // Context ID (opaque string)
  applicationId: string;// Installed application ID
  rootHash: string;     // Current state root hash
  members: string[];   // Identity public keys with access
}
